手把手教你实现阿里云DCDN全站加速Github Page

2025/4/16 Tech Experience

前期准备：

  1. 已经将网页部署在Github上（可使用xxxxx.github.io访问）

    可参考教程

  2. 有自己的域名（笔者使用的阿里云，方便管理，当然其他的也行）

  3. 已经申请了HTTPS证书

  4. 有足够的时间与耐心，享受探索本身的乐趣

  5. 善用AI与Google，教程不一定能解决你的问题，学会自己定位问题并使用工具

  6. 别忘了领阿里云300元学生优惠卷，可降低你的部署成本

购买资源：

  1. 打开 DCDN控制台 ，在页面找到资源包购买

  2. 购买 下行流量包与静态HTTPS请求包 ，加速地区默认国内

DCDN域名管理：

  1. 域名管理–>添加域名，加速域名替换成你的域名，可以为二级域名

    如（yourdomain.com, xxx.yourdomain.com）

  2. 下滑页面，新增源站信息，填入你的Github Page域名

  3. 完成后别急着关页面，后续还需要用到

域名解析

  1. 返回控制台主界面，复制绿色涂抹处的域名与红色涂抹处CNAME值

    !

  2. 在 DNS域名解析 中添加记录

DCDN域名配置：

  1. 返回刚才的DCDN域名管理界面，配置你的加速域名

Github配置：

  1. 进入你的网站仓库–> Setting –> Pages –> Custom domain

  2. 填入你的域名即可

一些小问题

  1. 网站在切换页面后，会重定向到原域名

    解决方法： 把_config.yml中的 url 改成新域名，重新部署

  2. 配置好后访问仍然很慢

    解决方法： 尝试把网站中的图片压缩，减小缓存大小

至此，你已经完成了网站的DCDN配置，你的网站会比之前访问的更快
