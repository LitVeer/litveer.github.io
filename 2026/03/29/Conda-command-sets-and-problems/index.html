Conda 机器学习环境配置命令集

以 myenv myenv2 环境名为例子说明，不包含驱动与cuda安装

  • 创建环境

    conda create -n myenv python=3.11 #不使用配置文件创建

    conda env create -f environment.yml #使用配置文件创建

    conda create -n myenv2 --clone myenv #复制已有环境

  • 激活/退出环境

    conda activate myenv	#激活

    conda deactivate myenv	#退出，回到base环境

  • 删除环境

    conda deactivate myenv #先退出当前环境
conda remove -n myenv --all #然后删除整个环境

  • 导出环境

    conda env export > environment.yml #本质快照，可保留构建版本，所有包信息，但不推荐用于不同机器的迁移
pip freeze > requirements.txt	#pip依赖

    conda env export --from-history > environment.yml #只保留手动conda安装的包，推荐用于不同机器的环境迁移
pip freeze > requirements.txt	#pip依赖

  • 安装包

    conda install xxxx	#解决包依赖一致性问题，推荐用于安装torch等涉及c++的包
pip install xxxx	#只安装当前包，不解决冲突，推荐用于tqdm等纯python包
参数说明
-n	#环境名，必选
--name

-p #环境路径，默认在conda安装目录，可选
--prefix
【环境配置核心原则】
  1. conda 负责底层（Python + CUDA + 核心库）
  2. pip 负责上层（纯Python依赖）
  3. 每个项目独立环境
  4. 迁移使用 environment.yml + requirements.txt
  5. 先验证 GPU，再开始训练

常见问题解决方法

  • 机器学习环境搭建流程

    参考链接：https://pytorch.org/get-started/previous-versions/

    # 1. 创建环境
conda create -n myenv python=3.10
conda activate myenv

# 2. 安装 PyTorch
conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia

# 3. 安装其他依赖（检查依赖是否有torch等，会破坏环境）
pip install -r requirements.txt

# 4. 验证（输出版本号与True即成功）
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
